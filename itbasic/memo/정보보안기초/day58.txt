1. 보안의 필요성과 3대 요소 
1) 보안의 필요성 
	조직이나 업무에 있어서 가장 중요한 것이 무엇인지를 생각하는 것 
	정보의 기밀을 유지하는 것  => 기밀성 
	시스템을 안정적으로 가동시키는 것 => 무결성 ? 가용성 ? 

2) 보안의 3대요소 
	기밀성(Confidentiality)	: 허가받은 사람만 정보에 접근할 수 있다
		기밀성이 침해 받는다 => 기밀 정보의 유출의 원인 
		기밀성 확보 기술 : 암호화, 인증기술 
	무결성(Integrity)	: 정보가 본래 상태에서 변조되지 않았고 신뢰할 만한 상태를 의미 
		무결성 침해 받는다 => 변조의 원인 
		무결성 확보 기술 : 해시, 디지털 서명, 메시지 인증코드, 체크섬, 무결성 검증도구 
	가용성(Availability) : 정보에 접근할 수 있는 사람은 언제든지 그 정보에 접근할 수 있다 
		가용성 침해받는다 => 공격대상에게 액세스를 집중시켜 과부하 상태에 빠지게 만드는 
			DoS/DDoS 서비스 방해 공격
		가용성 확보 기술 : 재해복구(DR:Disaster Recovery) & 백업

3) 보안의 추가 요소 
	ISO/IEC 270001 = 진정성, 책임성, 부인방지, 신뢰성
	* ISO (International Organization for Standardization) : 국제 표준화 기구 
	* IEC (Intergational Electrochnical Commission) : 국제 전기 표준회의 

	진정성(Authen Ticity) : 서명이나 인증을 통해 이용자가 맞다는 것과 데이터가 변조되지 않았음을 가리킴  
	책임성(Accountability) : 시스템에 남아있는 기록(로그)과 같은 증거를 사용하여 언제 어떤 일이 일어났는지를 추적할 수 있는 				것과 로그가 변조되지 않고 시스템의 움직임을 위한 장치가 갖춰져 있다는 것을 가리킴 
	부인방지(Non-repudation) : 발생현상이나 작성된 데이터를 나중에 없었던 일로 하지 않도록 하는 것을 가리킴 
	신뢰성(Reliability) : 정보시스템의 처리가 적절하며 모순없이 작동할 수 있는 구성임을 가리킴 

4) 보안을 확보하기 위해 필요한 것 
	무엇을 보호할 지, 어떤 보안을 생각할지를 명확히 한다 
	보호해야 할 것을 둘러싼 리스크를 확인한다 
	특정된 리스크를 어떻게 할 지 생각하고 대처한다 
	리스크를 대처한 후 상황을 정리하고 리스크 확인과 대처, 대처 후의 상황을 파악하는 사이클을 반복한다 

5) 인증과 인가 
	인증 : 이용자가 누구인지를 시스템이 식별하는 것 
	인가 : 이용자가 자신에게 허가된 처리나 서비스의 내용을 확인하는 것 
	
2. 암호
	어떤 데이터를 외부인이 읽지 못하도록 하는 수단

1) 송신자, 수신자, 도청자 
	앨리스(Alice) : 메시지를 전송하는 역할 
	밥(Bob) : 메시지를 수신하는 역할 
	이브(Eve) : 도청자(Eavesdropper) 소극적인 공격자 (사람뿐 아니라 장비나 시스템이 될 수 있음) 
		앨리스와 밥 사이에 이루어지는 통신을 도청하지만 통신중인 메시지를 수정하지는 못함 
	맬로리(Mallory) :  악의를 가진(Malicious) 적극적인 공격자 
		앨리스와 밥 사이에 이루어지는 메시지를 수정하고 자신의 메시지로 대체한 이전의 메시지를 재전송할 수 있다 
	트렌트(Trent) : 신뢰할 수 있는 중재자(Trusted Arbitrator) 중립적인 위치에 있는 제3자 
	빅터(Victor) : verifier, 의도된 거래나 통신이 실제로 발생했음을 검증할 때 등장 


- 송신자(Sender) : 어떤 사람이 다른 사람에게 정보를 보낼 때 보내는 사람을 의미 
- 수신자(Receiver) : 받는 사람을 의미 
- 메시지(Message) : 송신된 정보를 총칭 

2) 암호화와 복호화 
	평문을 외부인만 읽을 수 없는 상태로 만드는 것을 암호화 
	암호화된 데이터를 원래의 누구나 읽을 수 있는 데이터로 되돌리는 것을 복호화 
	
- 암호화를 하기 위해서 필요한 것? 
	암호알고리즘(암호의 구조), 키(변경이 가능한 부분)  

	수동적인 공격(기밀성침해) : 도청, 스니핑, 트래픽분석 
		=> 개인정보 유출, 계정정보 노출 등 
	능동적인 공격(무결성침해, 가용성침해) : 메시지 변조, 삽입공격, 삭제공격, 재생공격, 가장(Masquerade)
		=> 시스템 다운, 데이터 변조, 서비스 불능 
	
- 암호시스템(Cyptosystem) : 암호화와 복호화를 수행하는 시스템 

3) 기호적 표현 
	평문 M, P
	암호문 C
	암호알고리즘 E
	복호화알고리즘 D
	키 K

	C = E_k(P)	: 평문 P를 키 k로 암호화하여(E) 암호문 C를 얻는다		C=E(K, P)
	P = D_k(C)	: 암호문 C를 키 k로 복호화하여(D) 평문 P를 얻는다 	P=D)K, C)


* 암호화 기법 종류 
* 인증기관, 인증서, 전자서명 

3. 암호의 역사 
1) 고대암호
스키테일 암호 
	고대 스파르타, 막대기(같은 지름의 원통)에 감은 후 글자를 쓰고 다시 풀어서 특정 막대에만 맞게 읽히도록 만든 암호 
카이사르 암호 
	알파벳을 일정한 거리만큼 평행이동하는 치환 암호 
	A -> D(3칸밀기) 
전치(Transposition) / 치환(Substitution) 암호 
	전치암호 : 글자의 순서를 바꿈 
	치환암호 : 글자를 다른 문자로 바꿈

2) 중세~르네상스 암호 
비제네르 암호 
	다중 치환 암호,  키워드를 사용하여 평문을 여러 단계로 치환 
	19세기에 카시스키(Kasiski)공격으로 해독불가능한 암호가 깨짐 

3) 근대 암호 
에니그마 암호기 
	독일군 전기기계식 암호장치 
	2차 세계대전 당시 앨런튜링과 블렛츨리파크 팀이 해독 
셔넌의 정보이론 기반 암호학 
	현대 암호학의 수학적 기초 확립 
	혼돈(Confusion)과 확산(Diffusion) 개념 
	- 혼돈(Confusion) : 키와 암호문 사이의 관계를 복잡하게
	- 확산(Diffusion) : 평문의 통계적 특성이 암호문 전체에 퍼지게 

4) 현대암호
대칭키 암호 : 하나의 비밀키로 암호화/복호화 
	장점 : 속도 빠름, 단점 : 키 분배 어려움 
- DES(Data Encryption Stndard) 
	1977년 등장, 미군 표준 채택 
	키 길이가 56비트
 
- 3DES(Triple DES) 
	DES를 3번 반복 
	보안성 강화, 성능은 떨어짐 

- AES(Advanced Encryption Standard) 
	2001년, DES를 대체하는 암호 
	Rijndael알고리즘을 채택 
	128/192/256 비트 키 지원, 현재 국제 표준 

비대칭키 : 공개키와 개인키를 사용하는 방식 
	키 분배 문제를 해결

- RSA(비대칭키 암호)
	1977년 등장, Rivest, Shamir, Adleman이 개발 
	
- ECC(Elliptic Curve Cryptography) 
	타원 곡선 기반 
	RSA보다 짧은 키로 같은 수준의 보안을 제공함 
	
5) 단방향
- Hash Function  
	입력값 -> 고정된 길이의 해시값(다이제스트) 생성 
	데이터 무결성 검증에 사용 
	MD5 (128비트) : 현재 취약 
	SHA-1 (160비트) : 취약 
	SHA-2 (256/512비트) : 현재 안전한 표준 
	SHA-3	: 차세대 표준
